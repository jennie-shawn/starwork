# 主库同步

这里是本卫星项目与主库关系的可见说明。

主库不是本项目的上级工作目录，也不是项目进度总账本。主库只提供跨项目共享机制；本项目自己的状态、当前工作、参考资料、草稿和确认成果仍然留在本项目内。

## 来自主库的资源

| 本项目路径 | 来源 | 落地方式 | 默认规则 |
|---|---|---|---|
| `_系统/身份/` | 主库 `identity/` | 初始化快照或链接 | 默认只读；身份修正需要回写主库确认。 |
| `_系统/教训/` | 主库 `lessons/` + 本项目教训候选 | 初始化快照或本地沉淀 | 默认只读主库快照；新教训先留在项目内，确认可复用后再回写主库。 |
| `.starwork/internal/` | 主库 `.internal/` | 指定协议快照 | 默认只读；只放稳定协议，不放项目草稿。legacy 路径为 `.internal/`。 |
| `.obsidian/` | 主库 `.obsidian/` | 配置快照 | 可本地使用；通用配置变更优先回主库维护。 |
| `知识/` | 主库 `knowledge/` | 只读软链接 | 不在卫星项目内直接改共享知识。 |
| `.agents/skills/` | 主库正式 skills | 按需软链接 | 不复制、不改造主库 skill；需要改造时创建项目专用 skill。 |
| `.claude/skills/` | 主库正式 skills | 按需软链接 | 同上。 |
| `.starwork/sync.json` | 本项目生成 | 同步元数据 | 记录主库来源、同步时间、资源和 skill 挂载信息。legacy 路径为 `.core-sync.json`。 |

## 本项目自己的事实源

| 本项目路径 | 用途 |
|---|---|
| `_系统/上下文/当前项目.md` | 当前项目状态。主库只读取，不复制进 registry。 |
| `_系统/任务/当前工作.md` | 当前工作入口。 |
| `输出/确认成果/` | 默认确认成果位置；项目可以另行声明正式事实源。 |
| `参考资料/` | 本项目原始资料和参考材料。 |
| `.starwork/handoff/` | 跨项目联络单本地收发记录。 |

## 回写主库时怎么做

- 身份、教训、知识、通用规则这类可能跨项目复用的内容，不直接改主库正式目录。
- 先在本项目内形成候选内容，再按主库约定提交到主库 `.incoming/` 或通过跨项目联络单请求主库处理。
- 项目特定内容留在本项目，不回写主库。

## 为什么没有一个 `main-repo/` 文件夹

主库不是被复制进来的子目录。卫星项目只保留必要的快照、链接和同步元数据。

如果把所有主库内容塞进 `main-repo/`，用户会误以为可以在项目里直接维护主库；这会破坏主库和卫星项目的边界。

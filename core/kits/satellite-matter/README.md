# 卫星项目 Matter Kit

Preset: `satellite-matter`

适合接入主库、并启用事项机制的卫星项目。

## 包含

- 默认中文路径 profile
- 轻量输出模式
- 事项模式
- Decisions
- 主库同步模式
- 共享 skill 软链接挂载能力

## 规则

主库是规则源和共享资源源，卫星项目是执行层。

本 kit 预期包含：

- `_系统/主库同步/`：主库与卫星项目关系的可见说明
- `_系统/上下文/当前项目.md`：当前主库兼容的项目状态事实源
- `_系统/任务/当前工作.md`：当前工作入口
- `_系统/跨项目/`：跨项目联络单的本地收发记录
- `_系统/身份/`：来自主库的身份信息初始化快照或链接，默认只读
- `_系统/教训/`：来自主库的跨项目教训快照，以及项目内可复用教训候选
- `.internal/`：来自主库的选定协议文件快照
- `知识/`：指向主库 `knowledge/` 的只读链接
- `参考资料/`：本项目原始资料和参考材料
- `输出/`：本项目草稿和确认成果
- `事项/`：本项目事项推进过程
- `.core-sync.json`：记录主库来源、同步时间、共享资源和 skill 挂载信息
- `.agents/skills/`、`.claude/skills/`：按需软链接主库正式 skill

项目特定事实进入项目自己的正式事实源，不写回主库正式目录。

`_系统/上下文/当前项目.md` 是卫星项目的项目状态事实源。不要同时维护另一份 `_系统/上下文/项目状态.md`。

注意：`.core-sync.json`、`.internal/`、`.obsidian/` 是隐藏路径，很多文件管理器默认看不到。人类理解主库关系时，优先读 `_系统/主库同步/README.md`。

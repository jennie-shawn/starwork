# StarWork 工作区规则

## 开始前先读

1. `_系统/上下文/当前项目.md`
2. `_系统/任务/当前工作.md`
3. 涉及 Hub 资源、共享 skill 或跨项目同步时，读 `_系统/主库同步/README.md`

## 写入边界

- 项目状态写入 `_系统/上下文/当前项目.md`
- 当前工作入口写入 `_系统/任务/当前工作.md`
- 身份信息放入 `_系统/身份/`，默认只读
- 项目教训放入 `_系统/教训/`
- 独立项目知识入口放在 `知识/`；Satellite Project 默认把这里作为 Hub `knowledge/` 的只读入口
- 原始资料放入 `参考资料/`，默认只读
- AI 草稿放入 `输出/草稿/`
- 用户确认后的成果放入项目声明的正式事实源
- 跨项目请求、回执和交接的本地收发记录放入 `.starwork/handoff/`
- Hub 同步规则说明放在 `_系统/主库同步/`；不要创建一个复制 Hub 内容的 `main-repo/` 目录

## 跨项目联络

- 本项目的本地 inbox / outbox / sent / archived 在 `.starwork/handoff/`。
- Hub 的中央路由在 Hub 的 `projects/coordination/`。
- 跨项目联络不会直接改写目标项目的业务事实源；目标项目收到后自行判断、处理和回执。

## 需要确认

- 修改身份、教训、共享知识或 Hub 同步内容
- 将草稿晋升为正式事实源

# StarWork 工作区规则

## 开始前先读

1. `_系统/上下文/当前项目.md`
2. `_系统/任务/当前工作.md`
3. 如果存在 `.starwork/rules/index.md`，再按索引读取扩展规则

## 相关时再读

- 涉及用户偏好、沟通方式、领域背景或长期上下文时，读 `_系统/身份/README.md`
- 做重复性、风险较高或容易踩坑的工作前，读 `_系统/教训/README.md`
- 需要项目知识、资料索引或可复用参考时，读 `知识/README.md`
- 只有当工作台明确接入 Hub 时，才读取 Hub 同步说明

## 文件边界

- 项目状态写入 `_系统/上下文/当前项目.md`
- 当前执行记录写入 `_系统/任务/当前工作.md`
- 用户偏好、沟通方式和长期背景写入 `_系统/身份/README.md`
- 会改变未来行为的复用经验写入 `_系统/教训/README.md`
- 项目知识入口或知识索引放在 `知识/`
- 原始资料放入 `参考资料/`
- AI 草稿放入 `输出/草稿/`
- 用户确认后的成果放入项目声明的正式事实源
- StarWork 机制状态放入 `.starwork/`

## 工作方式

- 不把初始化日志、命令执行结果或临时解释写入项目事实源
- 不创建含义重叠的顶层目录；新增长期目录前先确认用途
- 如果 `.starwork/rules/` 中有扩展规则，按索引读取，不把规则文件当作项目进度

## 需要确认

- 修改身份或稳定偏好
- 将候选教训提升为稳定教训
- 将草稿晋升为正式事实源
- 改变工作台结构或顶层业务目录

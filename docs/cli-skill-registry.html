CLI + Skills Registry

能力注册表

这页是 StarWork 当前 CLI 与自研 Skills 的总览。后续新增命令、改名、能力边界变化，都要同步更新这里。

CLI负责稳定执行：创建、检查、适配、安装、升级。
Skills负责 AI 判断：采访、诊断、生成 blueprint。
Blueprint让 AI 的判断变成 CLI 可校验、可 dry-run 的施工图。
维护日期最后更新：2026-05-22。

当前总览

截至 alpha.7，StarWork 已有 7 个可用 CLI 命令和 6 个自研 Skill。`update` 和 `starwork skill` 仍是未来命令，不在当前可用列表里。

CLI

执行器

命令行负责可重复、可验证、低风险的文件操作。凡是涉及写入，都应支持 dry-run 或明确确认。

Skills

判断器

Skill 负责把用户的模糊需求变成方案、诊断或 blueprint。它们不直接替代 CLI，也不静默改文件。

状态

alpha.7

当前开发版本为 @jennie-shawn/starwork@0.1.0-alpha.7，已包含 Kit 自带 Skill、Hub 托管 Skill registry、项目 .starwork/skills.json 和 Agent Lanes 协作入口。

Skill 注册表

这里列出 StarWork 自研 Skill。Skill 负责 AI 判断和方案生成，不直接替代 CLI。

Skill 定位 触发场景 主要能力 产出 对应 CLI 状态
starworkInit 初始化设计助手。 用户想从 0 建工作台，但还没确定工作台类型、语言、事项机制或目录偏好。 分支式采访；判断 Hub / 单项目；判断语言；判断是否需要事项；生成初始化建议或 init blueprint。 初始化建议、未来 init blueprint、执行命令建议。 starwork init 已实现
starworkSpawn Hub Kit 自带的卫星项目定制单设计助手。 用户已有 Hub，要创建一个新 Satellite 项目，并希望项目目录、规则或托管 Skill 可定制。 采访项目类型和工作方式；判断 starter / matter；生成 spawn blueprint；整理项目规则和 seed。 blueprint.json、rules、seed、spawn dry-run 命令。 starwork spawn --blueprint 已实现
neat-freak 单项目 Kit 自带的整理收尾助手。 单项目工作台需要阶段性清理、项目记忆整理、收尾归档。 读取工作区规则和当前工作；检查产品事实、过程记录和当前任务是否归位；输出清理报告。 清理报告、必要的当前工作记录更新建议。 starwork init 随 Kit 写入 已实现
starworkDoctor 工作区诊断助手。 用户要理解 doctor 输出、判断旧目录和 StarWork Core 逻辑是否接近、整理升级建议。 读取 doctor JSON；解释 inventory 和 signals；建立 Core 角色映射；判断 Core 逻辑贴近程度；提出整理建议。 诊断报告、目录角色映射、缺失和风险、建议确认的问题。 starwork doctor 已实现
starworkUpgrade 升级蓝图设计助手。 用户已有旧模板或非标准目录，想无损升级成 StarWork 工作台。 基于 doctor 和 starworkDoctor 诊断采访用户；确认目录语义；选择升级策略；生成 upgrade blueprint 和规则文件。 upgrade-blueprint.json、rules、dry-run 命令。 starwork upgrade --blueprint 已实现
starworkMultiagent 多 Agent 协作命令包装助手。 用户想把当前会话登记为常用智能体、管理 Agent 职责位、绑定或释放 session、查看分工、登记共享输出。 判断用户语义；确认 lane ID、职责、写入范围和 session；把请求翻译成 multiagent init/add/bind/release/status/share 命令组合。 multiagent dry-run 命令、确认后的执行命令、当前分工解释。 starwork multiagent 已实现

配合关系

最容易混淆的是 doctor / upgrade 这一组，以及 multiagent 的用户语义和 CLI 子命令。这里固定它们的链路。

01 探测

starwork doctor --json 只暴露事实、inventory、signals 和 checks。

02 诊断

starworkDoctor 判断当前目录和 StarWork Core 工作逻辑的贴近程度。

03 设计

starworkUpgrade 采访用户，生成可执行的升级蓝图。

04 执行

starwork upgrade --blueprint 只按蓝图执行低风险动作。

固定边界：CLI 不硬猜业务语义；Skill 不静默改用户文件。所有会写入的 CLI 能力都应保留 dry-run 或明确确认。multiagent init 是协议初始化；“登记当前会话为常用智能体”在 Skill 层应解释为必要时先 init，再 add 和 bind。

常用命令

这些命令用于 A 测、演示和本地回归测试。

初始化

starwork init --type single-light --pack general --target ./my-workspace --dry-run
starwork init --type hub --target ./my-hub --yes

从 Hub 创建项目

starwork spawn --hub ./my-hub --name "新项目" --target ./new-project --mode matter --dry-run
starwork spawn --hub ./my-hub --target ./new-project --blueprint ./blueprint.json --dry-run

检查与升级

starwork doctor --target ./old-workspace --json --inventory-depth all
starwork upgrade --target ./old-workspace --blueprint ./upgrade-blueprint.json --dry-run

适配与安装 Pack

starwork adapt claude --target ./my-workspace --yes
starwork pack install content-creator --target ./my-workspace --dry-run

多 Agent 协作

starwork multiagent init --lanes research,writing,review --target ./my-workspace --yes
starwork multiagent bind research --session codex:manual-research-1 --target ./my-workspace --yes
starwork multiagent share research --title "Source summary" --path product/docs/source-summary.md --audience writing,review --target ./my-workspace --yes

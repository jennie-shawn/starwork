CLI + Skills Registry

能力注册表

这页是 StarWork 当前 CLI 与自研 Skills 的总览。后续新增命令、改名、能力边界变化，都要同步更新这里。

CLI负责稳定执行：创建、检查、适配、安装、升级。
Skills负责 AI 判断：采访、诊断、生成可执行方案。
Blueprint让 AI 的判断变成 CLI 可校验、可 dry-run 的施工图。
维护日期最后更新：2026-05-22。

当前总览

截至 alpha.13，StarWork 已有 9 个可用 CLI 命令和 6 个自研 Skill。`update` 和 `starwork skill` 仍是未来命令，不在当前可用列表里。新增或调整命令、skill、Kit 分发范围、npm 版本时必须同步维护本页。

CLI

执行器

命令行负责可重复、可验证、低风险的文件操作。凡是涉及写入，都应支持 dry-run 或明确确认。

Skills

判断器

Skill 负责把用户的模糊需求变成方案或诊断。它们不直接替代 CLI，也不静默改文件。

配合关系

最容易混淆的是 doctor / upgrade 这一组，以及 multiagent 的用户语义和 CLI 子命令。这里固定它们的链路。

01 探测

starwork doctor --json 只暴露事实、目录清单、线索和检查结果。

02 诊断

starworkDoctor 判断当前目录分工是否清楚，以及如何整理成 StarWork 工作台。

03 设计

starworkDoctor 在用户确认后生成可执行的升级方案。

04 执行

starwork upgrade --blueprint 只按方案执行低风险动作。

固定边界：CLI 不硬猜业务语义；Skill 不静默改用户文件。所有会写入的 CLI 能力都应保留 dry-run 或明确确认。multiagent init 是协议初始化；“登记当前会话为常用智能体”在 Skill 层应解释为必要时先 init，再 add 和 bind。

常用命令

这些命令用于 A 测、演示和本地回归测试。

初始化

starwork init --type project --pack general --target ./my-workspace --dry-run
starwork init --type hub --target ./my-hub --yes

从 Hub 创建项目

starwork spawn --hub ./my-hub --name "新项目" --target ./new-project --mode project --language zh --dry-run
starwork spawn --hub ./my-hub --target ./new-project --blueprint ./blueprint.json --dry-run

检查与升级

starwork doctor --target ./old-workspace --json --inventory-depth all
starwork upgrade --target ./old-workspace --blueprint ./upgrade-blueprint.json --dry-run

适配与安装场景能力

starwork adapt claude --target ./my-workspace --yes
starwork pack install content-creator --target ./my-workspace --dry-run

多 AI 协作

starwork multiagent init --lanes research,writing,review --target ./my-workspace --yes
starwork multiagent bind research --session codex:manual-research-1 --session-name "Research Agent" --target ./my-workspace --yes
starwork multiagent share research --title "Source notes draft" --path notes/source-notes.md --audience writing,review --target ./my-workspace --yes

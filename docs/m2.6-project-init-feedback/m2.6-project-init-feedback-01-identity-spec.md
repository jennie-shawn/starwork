# M2.6 Project Init 反馈 01：Identity 模板口径 SPEC

## 状态

- 版本：v0.1 implemented
- 所属阶段：M2.6 既有功能优化与 A 测稳定化
- 所属模块：StarWork Core / Project Kit / Profiles / CLI init
- 反馈来源：单项目初始化 A 测
- 问题类型：Core 架构和模板内容错误

## 用户反馈

单项目创建完成后，`_system/identity/README.md` 写成了类似 Hub identity snapshot 的说明。用户当前创建的是独立单项目工作台，不知道这里应该写什么，也会误以为 identity 必须来自 Hub 或主库。

## 根因

Project Kit 当前同时承担两种形态：

- 独立 Project：用户从 0 创建的单项目工作台。
- Satellite Project：从 Hub 派生并受 Hub 管理的项目。

模板和英文映射逻辑没有按形态区分 identity 口径，导致独立 Project 也继承了 Satellite 的“Hub 快照 / 只读同步”叙述。

相关风险不止 identity。任何 `_system/identity/`、`_system/lessons/`、`knowledge/`、`.agents/skills/`、`.claude/skills/` 的说明，只要把 Hub 作为默认来源，都会让独立 Project 用户误解。

## 目标

独立 Project 的 identity 必须是“项目本地长期上下文模板”，而不是 Hub 同步说明。

用户和 AI 打开 identity 后，应明确知道：

- 这里能写什么。
- 什么场景下要读。
- 什么内容需要用户确认后才能改。
- 它与 Hub/Satellite 同步没有默认关系。

## 非目标

- 不在本 SPEC 里实现完整用户画像系统。
- 不要求初始化时强迫用户填写 identity。
- 不把 identity 变成全局账户、登录、云同步或商业系统。
- 不删除 Satellite 场景对 Hub identity snapshot 的支持。

## 目标结构

独立 Project 初始化后，identity 目录至少应包含一个可填写模板：

```text
_system/identity/
└── README.md
```

中文镜像：

```text
_系统/身份/
└── README.md
```

README 必须像模板，不只是说明。

建议内容结构：

```markdown
# Identity

## Durable Context

- Project owner / user:
- Working style:
- Domain background:
- Long-lived preferences:

## Communication Preferences

- Preferred language:
- Tone:
- Detail level:

## Stable Constraints

- Do:
- Avoid:

## Update Rule

Read this file when user preference, domain background, or long-term context matters.
Ask before changing stable identity or preferences.
```

中文对应：

```markdown
# 身份

## 长期背景

- 项目负责人 / 用户：
- 工作习惯：
- 领域背景：
- 长期偏好：

## 沟通偏好

- 常用语言：
- 语气：
- 细节程度：

## 稳定约束

- 应该：
- 避免：

## 更新规则

当任务涉及用户偏好、领域背景或长期上下文时读取本文件。
修改稳定身份和偏好前必须确认。
```

## 形态分流规则

### 独立 Project

`starwork init --type project` 生成的工作台：

- identity 是项目本地长期上下文。
- 默认可作为候选信息维护。
- 不出现 “Hub identity snapshot” 作为默认描述。
- 不出现 “main repo” 或 “主库分发” 作为默认前提。

### Satellite Project

`starwork spawn` 从 Hub 生成的项目：

- identity 可以说明来自 Hub 的快照或只读链接。
- 必须同时说明项目本地候选更新如何回写 Hub。
- 该说明只在 workspace state 存在 `hub` 字段时成立。

## 举一反三排查

实现前必须检查这些位置，不能只改一个 README：

- `product/core/kits/project/_系统/身份/README.md`
- `product/core/kits/project/_系统/教训/README.md`
- `product/core/kits/project/知识/README.md`
- `product/core/profiles/en/reference-kits/local-starter/_system/identity/README.md`
- `product/core/profiles/en/reference-kits/local-starter/_system/lessons/README.md`
- `product/cli/src/cli.js` 中 `renderSatelliteKitContent()` 对 identity / lessons / knowledge 的英文覆盖文案
- `product/cli/src/cli.js` 中 `buildInitPlan()` 与 `buildSpawnPlan()` 是否区分独立 Project 和 Satellite Project
- `product/core/kits/project/AGENTS.md` 和英文映射后的 `AGENTS.md`

排查关键词：

```text
Hub identity
main repo
主库分发
初始化快照
read-only link
snapshot
synced main-repository content
```

这些词可以存在于 Satellite / Hub 同步说明里，但不能作为独立 Project identity 的默认口径。

## 验收标准

### 文件验收

- `starwork init --type project --language en --pack general --target <tmp> --yes` 后，`_system/identity/README.md` 是可填写模板。
- 该文件不包含 `Hub identity snapshot`、`main repo`、`synced main-repository`、`satellite` 等默认同步口径。
- `starwork init --type project --language zh --pack general --target <tmp> --yes` 后，`_系统/身份/README.md` 是中文可填写模板。
- 中文 identity 文件不包含“主库分发”“初始化快照或链接”作为默认说明。

### 行为验收

- `starwork doctor --target <tmp>` 对中英文独立 Project 均通过。
- `starwork spawn --hub <hub> ...` 生成的 Satellite 仍可保留 Hub identity snapshot 说明。
- `doctor --json` 能通过 workspace state 区分独立 Project 与 Satellite Project，不把独立 Project 标记为缺少 Hub identity。

### AI 使用验收

给 Agent 只读 `AGENTS.md` 和 identity README 后，Agent 应能回答：

- identity 用来记录用户长期偏好和项目背景。
- 涉及偏好、背景、沟通方式时应该读取 identity。
- 修改 identity 前需要用户确认。

### 防复发验收

- 新增或更新测试覆盖中英文 `init --type project` 的 identity 文案。
- 测试必须断言独立 Project identity 不含 Hub/Satellite 同步词。
- 测试必须断言 Satellite identity 可以含 Hub 来源说明。
- 未来修改 Project Kit、英文映射或 `renderSatelliteKitContent()` 时，相关测试必须继续通过。

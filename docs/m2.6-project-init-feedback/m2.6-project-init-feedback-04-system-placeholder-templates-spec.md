# M2.6 Project Init 反馈 04：System 占位模板 SPEC

## 状态

- 版本：v0.1 implemented
- 所属阶段：M2.6 既有功能优化与 A 测稳定化
- 所属模块：StarWork Core / Project Kit / Profiles
- 反馈来源：单项目初始化 A 测
- 问题类型：Core 模板可用性不足

## 用户反馈

`_system/identity/` 里只有 README，用户和 AI 都不知道下面应该写什么。System 文件夹下的文件应该是模板或占位文档，而不是纯说明。

## 根因

当前 Project Kit 把 `_system/` 当成机制说明层维护，而不是当成用户和 Agent 会长期使用的工作入口。

README 只能解释“这个目录是什么”，但不能稳定指导“这里该写什么”。对 AI 来说，缺少字段模板会导致两种问题：

- 不写，长期上下文缺失。
- 自由发挥，写入不稳定结构。

## 目标

System 层每个用户可写或候选可写的区域，都要有清晰占位模板。

用户打开后应该能填写；AI 读取后应该知道结构；doctor 或测试应该能判断模板是否存在。

## 非目标

- 不要求用户初始化时填写所有模板。
- 不让 AI 自动补全用户身份或教训。
- 不把 system 变成复杂数据库。
- 不在本 SPEC 中设计 UI 表单。

## System 文件分类

### 事实源文件

必须有明确字段：

```text
_system/context/current-project.md
_system/tasks/current-work.md
```

### 长期上下文目录

至少 README 本身必须是可填写模板：

```text
_system/identity/README.md
_system/lessons/README.md
```

### 机制说明目录

只在相关形态出现：

```text
_system/main-repo-sync/README.md
```

独立 Project 默认不出现。

## 建议模板

### current-work

```markdown
# Current Work

## Now

- TBD.

## Next

- TBD.

## Waiting On

- TBD.

## Notes For Next Agent

- TBD.
```

### identity

见 `m2.6-project-init-feedback-01-identity-spec.md`。

### lessons

```markdown
# Lessons

Record reusable lessons that should change future behavior.

## Active Lessons

- TBD.

## Candidate Lessons

- TBD.

## How To Add A Lesson

A good lesson should be specific, reusable, and behavior-changing.
Do not record ordinary progress summaries here.
Ask before promoting a candidate lesson into a stable lesson.
```

中文：

```markdown
# 教训

记录会改变未来行为的可复用经验。

## 已确认教训

- 待填写。

## 候选教训

- 待填写。

## 写入规则

好的教训应该具体、可复用，并能改变后续行为。
不要把普通进度摘要写到这里。
将候选教训提升为稳定教训前需要确认。
```

### knowledge

独立 Project 默认 `knowledge/README.md` 应说明：

- 存放项目本地知识入口或索引。
- 不默认是 Hub 链接。
- 若未来连接 Hub，再转为只读共享知识入口。

## 举一反三排查

排查所有只有说明、没有可填写结构的 system 文件：

- `_system/context/current-project.md`
- `_system/tasks/current-work.md`
- `_system/identity/README.md`
- `_system/lessons/README.md`
- `knowledge/README.md`
- 中文镜像下所有对应文件
- `core/profiles/en/reference-kits/local-starter/**`
- `core/profiles/zh/templates/**`
- CLI 英文覆盖函数 `renderSatelliteKitContent()`

排查关键词：

```text
Put ... here.
This folder contains ...
TBD
待填写
read-only by default
snapshot
candidate
```

原则：

- 只有机制说明目录可以纯说明。
- 用户会写的目录必须有字段模板。
- 只读目录必须说明何时读、何时不能改。

## 验收标准

### 文件验收

中英文 `starwork init --type project` 后：

- current-project 存在，并包含 Goal/Stage/Focus/Risks/Next 或中文等价字段。
- current-work 存在，并包含 Now/Next/Waiting On/Notes For Next Agent 或中文等价字段。
- identity README 存在，并包含 Durable Context/Communication Preferences/Stable Constraints 或中文等价字段。
- lessons README 存在，并包含 Active Lessons/Candidate Lessons/How To Add A Lesson 或中文等价字段。
- knowledge README 对独立 Project 不写成 Hub 只读链接。

### AI 验收

只给 Agent system 目录后，Agent 应能判断：

- 当前项目状态写 current-project。
- 当前推进写 current-work。
- 用户偏好和长期背景写 identity。
- 可复用经验写 lessons。
- 普通进度不能写 lessons。

### Doctor 验收

`starwork doctor` 至少检查：

- system 事实源文件存在。
- identity 和 lessons 入口存在。
- 如果未来新增 schema，可检查必需标题。

### 防复发验收

- 新增 snapshot 或文本断言测试，覆盖中英文 Project 初始化模板。
- 测试不得只检查文件存在，还要检查关键标题。
- 后续新增 system 目录时，必须同时定义“读写规则”和“占位模板”。

# M2.6 Project Init 反馈 03：Current Project 事实源纯度 SPEC

## 状态

- 版本：v0.1 implemented
- 所属阶段：M2.6 既有功能优化与 A 测稳定化
- 所属模块：StarWork Core / CLI init / Skills / Blueprint
- 反馈来源：单项目初始化 A 测
- 问题类型：Skill 输出污染项目事实源

## 用户反馈

`_system/context/current-project.md` 里出现了 “Initialized as a StarWork project workspace with a custom code-focused structure.” 这类 StarWork 初始化过程描述。用户要的是自己的项目状态，而不是 StarWork 施工记录。

## 根因

`current-project.md` 既被模板当成项目事实源，又被 Agent/blueprint 当成初始化结果摘要的承载位置。

在 Skill 约束不足时，AI 会把以下内容写进项目事实源：

- StarWork 初始化完成情况。
- blueprint 定制说明。
- 未使用目录列表。
- CLI 施工步骤。
- “下一步运行 doctor” 这类工具建议。

这些内容属于 `.starwork/` 机制层、CLI 输出或会话回复，不属于用户项目事实。

## 目标

`current-project.md` 必须只记录用户项目事实。

它可以记录：

- 项目目标。
- 当前阶段。
- 近期重点。
- 主要事实源。
- 当前风险。
- 下一步业务动作。

它不能记录：

- StarWork 初始化日志。
- blueprint 文件路径。
- CLI dry-run 结果。
- 哪些默认目录没有创建。
- npm / skills / doctor 安装验证过程。
- Agent 自己的施工解释。

## 非目标

- 不禁止 `.starwork/workspace.json` 记录创建来源。
- 不禁止 CLI 在终端输出初始化总结。
- 不禁止 Agent 在对话里向用户汇报执行结果。
- 不要求 current-project 在初始化后就完整填写。

## 模板要求

独立 Project 默认 `current-project.md` 应是干净占位模板：

```markdown
# Current Project

## Goal

TBD.

## Current Stage

TBD.

## Focus

- TBD.

## Primary Sources

- TBD.

## Risks

- TBD.

## Next Step

- TBD.
```

中文对应：

```markdown
# 当前项目

## 目标

待填写。

## 当前阶段

待填写。

## 近期重点

- 待填写。

## 主要事实源

- 待填写。

## 风险

- 待填写。

## 下一步

- 待填写。
```

## Blueprint 写入边界

`init-blueprint.json` 不应直接生成或覆盖 `current-project.md`，除非 schema 明确新增专门字段，例如：

```json
{
  "project_context": {
    "goal": "...",
    "stage": "...",
    "focus": ["..."],
    "primary_sources": ["src/", "docs/"]
  }
}
```

即便支持该字段，也必须满足：

- 只写用户项目事实。
- 不写 StarWork 初始化过程。
- 不写 CLI 施工信息。
- 不写 Agent 自我评价。

## Skill 防发挥规则

`starworkInit` 必须明确：

- 创建工作台时，可以把执行结果回复给用户，但不要把执行结果写进 `current-project.md`。
- 如果用户没有提供项目目标，`current-project.md` 保持 TBD。
- 如果用户提供的是“我要创建一个代码项目，代码放 src，文档放 docs”，可以写入 primary sources，但不要写“Initialized as StarWork”。
- “Folders Not Used”“不要创建 outputs/references” 属于规则或 blueprint 约束，不属于 current project。

## 举一反三排查

排查所有可能被 AI 污染的项目事实源：

- `_system/context/current-project.md`
- `_系统/上下文/当前项目.md`
- `_system/tasks/current-work.md`
- `_系统/任务/当前工作.md`
- `README.md`
- `AGENTS.md`
- `.starwork/rules/*.md`

排查关键词：

```text
Initialized as
StarWork project workspace
blueprint
dry-run
doctor
Folders Not Used
not used
generated by starwork
created by starwork
```

允许这些词出现在：

- `.starwork/workspace.json`
- CLI stdout
- 测试断言
- SPEC 文档

不允许默认出现在：

- `current-project.md`
- `AGENTS.md`
- 用户业务 README

## 验收标准

### 文件验收

执行带定制 blueprint 的初始化后：

```bash
starwork init --target <tmp> --blueprint <init-blueprint.json> --yes
```

`_system/context/current-project.md` 必须满足：

- 不包含 `Initialized as`。
- 不包含 `StarWork project workspace`。
- 不包含 `blueprint`。
- 不包含 `Folders Not Used`。
- 不包含 `doctor` 作为初始化步骤。
- 如果用户没有提供明确项目事实，相关字段保持 `TBD`。

### 行为验收

- `doctor` 仍通过。
- CLI stdout 可以提示 `doctor` 下一步，但不写入 current-project。
- `.starwork/workspace.json` 可以记录 `created_by` 和 customization。

### Skill 验收

给 `starworkInit` 一个需求：

```text
Create an English code project. Use src for code and docs for design docs. Do not use references or outputs.
```

Agent 生成的 current-project 只能写：

- Goal: code project 或用户真实项目目标。
- Primary Sources: `src/`, `docs/`.

不得写：

- StarWork 初始化状态。
- 未使用目录列表。
- CLI 执行过程。

### 防复发验收

- 新增测试读取初始化后的 current-project，断言不含污染关键词。
- `starworkInit/SKILL.md` 必须包含“项目事实源纯度”约束。
- 如果未来新增 `project_context` blueprint 字段，必须有 schema 校验和测试覆盖。
